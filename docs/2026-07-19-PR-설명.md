### 2. 초대코드 SHA-256 → 평문 전환 (버그 수정)
- `crypto.subtle`은 secure context(https·localhost) 밖에서만 존재 → 로컬 http 등에서 해시 검증이 **조용히 실패**(맞는 코드도 안 넘어가고 경고도 안 뜸). 정적 사이트라 해시는 원래 실질 보안이 아니었으므로(README 명시) 평문 `code` 비교로 전환.
- 게이트 판단을 `index.json`의 `locked` 기준으로 격상 → 데이터/캐시 불일치에도 잠금 일정이 그냥 열리지 않는 **fail-safe**.
- 오답 시 빨간 경고(`❌ 초대 코드가 맞지 않아요`) 표시, 재입력 시 경고 해제.
- ⚠️ **보안 주의**: 코드가 JSON에 평문으로 보임. 원래도 정적 사이트라 데이터는 공개이며 "가벼운 잠금"임 — 민감 정보 금지.

### 3. 스키마 불일치 크래시 수정
