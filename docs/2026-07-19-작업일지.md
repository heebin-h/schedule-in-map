### 2-3. 초대코드 평문 전환 (버그 근본수정)
- **증상**: 로컬 http에서 맞는 코드도 "안 넘어감", 오답 경고도 안 뜸.
- **근본원인**: `crypto.subtle`은 secure context(https·localhost) 밖에서 없음 → `sha256()` 예외 → `submitCode`가 미포착 → 조용한 실패. (`.err`은 이미 `var(--red)`로 빨강이므로 경고 미표시 = 검증 함수가 죽었다는 증거.)
- **조치**: SHA-256(`codeHash`) 제거 → 평문 `code` 비교. 정적 사이트라 해시는 원래 실질 보안 아님(README 명시). 대상: `js/data.js`, `js/viewer.js`, `js/editor.js`, 트립 JSON 2개, `README.md`.
- 참고: 기존 코드 복구값 — 서울 `1234`, 홍대 `0725`(= 방문일).

### 2-4. "비번 없이 열림" fail-safe化
- **원인**: 브라우저가 옛 `viewer.js`(=`trip.codeHash` 검사) 캐시 → 새 JSON엔 codeHash 없어 통과. (또는 `?b=` 원격 스테일 데이터.)
- **조치**: 게이트 근거를 `index.json`의 `locked`로 격상(`t.locked || trip.code`), `verifyCode`는 코드 일치만(잠금인데 코드 없으면 안전하게 닫힘). 오답 시 `❌ 초대 코드가 맞지 않아요`, 재입력 시 해제.
