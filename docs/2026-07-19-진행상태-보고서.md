- **핵심 문제: `robotgo-hongdae` 일정은 지금 열면 크래시한다.** 데이터 파일은 `places` 키를 쓰는데 뷰어 코드는 `trip.stops`만 읽음 → `undefined.map()` TypeError. → §3

- `index.json`은 `"locked": false`인데 데이터 파일엔 `codeHash`가 있어 **목록엔 잠금 표시가 없는데 열면 코드를 요구**한다. → §4
- 정상 동작하는 건 `seoul-day` 일정 하나뿐. 프로젝트 골격(정적 웹앱, DB 없음, 파일=데이터)은 견고함. → §2

### ⚠️ 핵심 버그 — `robotgo-hongdae` 일정 크래시

`data/trips/robotgo-hongdae.json:5`는 배열 키가 **`places`**, 항목 필드가 `{time, name, address, lat, lng, url}`.
반면 렌더 코드는 전부 **`trip.stops`** 와 `{name, time, memo}` 를 읽음:

- `js/viewer.js:108` `renderRoute(map, trip.stops, …)` → `trip.stops`가 `undefined`
- `js/map.js:17` `stops.map(s => …)` → **TypeError: undefined.map** → 뷰어 크래시
- `js/viewer.js:114,134` 도 `trip.stops[...]` 참조

결과: 목록에서 이 일정 선택 → 초대코드 통과 → **지도 화면이 뜨지만 빈 화면 + 콘솔 에러**. `address`/`url` 필드도 렌더 경로가 없어 무시됨.

> 검증: 코드 트레이스로 확정 (브라우저 런타임 실행은 미실시 — 확인법: 로컬 서버 띄우고 해당 일정 열어 콘솔 확인).

### ⚠️ 데드 CSS

`css/viewer.css` 끝에 추가된 `.address`, `.map-link`, `.map-link a`, `.map-link a:hover` (22줄)를 참조하는 JS/HTML이 없음 (`grep address|map-link js *.html` → 0건). codex가 주소·네이버 링크를 보여주려 CSS·데이터는 넣었지만 **뷰어 JS를 배선하지 않아** 절반만 완성된 상태.

## 4. 부수 문제 (correctness, 낮음~중간)

1. **locked 플래그 불일치** — `data/index.json:13`은 `robotgo-hongdae`를 `"locked": false`로 두는데, `data/trips/robotgo-hongdae.json:4`엔 `codeHash`가 있음. 목록엔 "초대 코드" 배지 없음(뷰어 `js/viewer.js:58`은 index의 `locked`로 배지 표시) → 그런데 `js/viewer.js:74`는 trip의 `codeHash`로 코드 화면을 띄움. **배지 없이 갑자기 코드 요구.**
2. **날짜 표기 불일치** — index `"2026. 7. 25-26"` (`index.json:12`) vs trip `"2026.07.25-26"` (`robotgo-hongdae.json:3`). 목록과 뷰어 헤더에 서로 다른 문자열이 뜸.
3. **스키마 문서 이탈** — `README.md`와 editor가 만드는 정본은 `stops`. codex가 손으로 쓴 `places` 파일만 규격 밖. editor.html로 다시 만들면 규격을 지킴.
